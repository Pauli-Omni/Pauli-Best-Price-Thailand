2. Geschenk-Overlay bei osg_vip-Scan
------------------------------------
Locale-Packs (Kultur-Persönlichkeit): assets/locales/
  de.json · en.json · th.json · zh.json · ru.json · pl.json
  Polnisch: vipGiftBodyFromTplMale/Female (Pauli podarował / Wii podarowała).
  osgVipWithLocalePackReady + osgBootAppLang laden pl.json bei Systemsprache pl.
  Schlüssel vipCultureGreetTts / vipCultureGreetBubble — Avatar-Spruch beim ersten VIP-QR-Scan.

Ablauf:
  1. Nutzer öffnet Link mit ?osg_vip=CODE oder löst Code manuell ein
  2. POST /api/vip/redeem → { ok, profile, gift }
  3. Bei gift.giftThb > 0: Overlay (Thai-Text) BEVOR osgVipActive gesetzt wird
  4. Button „รับของขวัญและเริ่มใช้งาน“ → Vollzugriff (VIP aktiv)

Berechnung (server.js: vipGiftFromProfile):
  - Einheit = inviteCascadePerFriend (falls > 0), sonst inviteFreeCount
  - giftThb = Einheiten × 59 THB

Beispiele (dynamisch im Overlay):
  | Situation                         | THB  | Thai (vipGiftBodyTpl / vipGiftBodyFromTpl)        |
  |-----------------------------------|------|---------------------------------------------------|
  | 1 Einladung (Slots 01–30)         | 59   | คุณได้รับของขวัญ 59 บาท!                          |
  | 2 Zugänge / Kaskade (55, 56)      | 118  | Pauli/Wii มอบของขวัญ 118 บาทให้คุณแล้ว!            |
  | 3 Einladungen (Familie 51–54)     | 177  | คุณได้รับของขวัญ 177 บาท!                          |
  | Influencer 31–50 (keine Einlad.)  | —    | kein Overlay                                      |

Deutsche Referenz (nur zur Erläuterung, Anzeige immer Thai):
  · „Dir wurden 59 THB geschenkt!“
  · „Pauli hat dir gerade 118 THB geschenkt!“ / „Wii hat dir …“

Client: index.html — osgBootAppLang, osgLoadLocaleOverlay, osgConsumeVipParamOnce,
osgVipShowGiftOverlay (Texte aus T[lang] nach JSON-Merge, UI-Sprache des Nutzers)

Geschenk-Overlay je Sprache (Beispiele):
  de: „Dir wurden {AMOUNT} THB geschenkt!“ / „{SENDER} hat dir gerade {AMOUNT} THB geschenkt!“
  en: „You received a gift of {AMOUNT} THB!“ / „{SENDER} just gifted you {AMOUNT} THB!“
  th: „คุณได้รับของขวัญ {AMOUNT} บาท!“ / „{SENDER} มอบของขวัญ {AMOUNT} บาทให้คุณแล้ว!“

Absender-Erkennung (server.js vipGiftFromProfile + Client):
  | Slot    | giftSenderDisplay        |
  |---------|--------------------------|
  | 55      | Wii                      |
  | 56      | Pauli                    |
  | 51–54   | Label (Mama, Schwester …)|
  | 01–30   | — (nur Betrag)           |

3. VIP-Provision pro Kauf-Event (02_Quellcode/Core_Logik/server.js)
-------------------------------------------------------------------
vipBuildProfiles → commissionPerPurchaseThb:

  | Slot    | Rolle           | THB / Kauf (aktuell) | früher |
  |---------|-----------------|----------------------|--------|
  | 31–40   | influencer      | 1,00                 | 0,50   |
  | 41–50   | vip_influencer  | 5,00                 | 1,00   |
  | übrige  | —               | 0                    | 0      |

Erfassung: POST /api/vip/event mit eventType=purchase → commissionThb in vip_events.jsonl
